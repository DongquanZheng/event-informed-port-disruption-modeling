# Project Log

## 2026-04-30

### Structure Decision

Created a separate GitHub-ready project folder named `event_augmented_port_disruption`.

### Research Decision

The project will predict future abnormal port activity rather than same-day abnormal activity. This keeps the role of news event signals predictive rather than merely descriptive.

### Method Decision

The first NLP module will use transparent rule-based event extraction from raw news titles and text. This is suitable for an MVP because the resulting event categories and risk scores are interpretable.

### Data Decision

The initial project uses sample CSV files generated by script. The code is structured so that real PortWatch-style data and real news data can replace the sample files later.

### Implementation Decision

Rolling operational and event features are calculated within each port or location group. This avoids accidental leakage across ports or countries in panel-style data.

### Engineering Decision

Pipeline scripts are designed to run from the project root with commands such as `python scripts/run_mvp_pipeline.py`.

### Sanity-Check Result

The MVP pipeline runs successfully on generated sample data. In this synthetic setting, event-augmented models outperform operational-only models on PR-AUC and F1. This is only a pipeline sanity check, not an empirical research conclusion.

### Workflow Correction

The project will be developed notebook-first. The `src/` modules and scripts are now treated as a later engineering scaffold, while the main teaching workflow starts from step-by-step Jupyter notebooks.

### Notebook Update

### Weekly Baseline Decision

The modeling unit was changed from day-level to week-level to better align PortWatch activity with sparse and noisy news event signals. Rotterdam daily PortWatch data were aggregated into 380 complete weekly observations from 2019-01-07 to 2026-04-13.

### Weekly Target Decision

The weekly target is `abnormal_activity_next_week`, defined as 1 when next week's port calls fall below the previous 8-week mean minus 1.5 standard deviations. This stricter definition was selected over quantile-based alternatives because it yields a more disruption-like positive rate of approximately 13%.

### Weekly Baseline Result

Using only weekly operational features, Logistic Regression achieved ROC-AUC 0.859, PR-AUC 0.591, F1 0.516, recall 0.889, and precision 0.364 on the temporal test set. Random Forest under the default 0.5 threshold did not identify positive cases, so its threshold behavior requires additional inspection before drawing conclusions.

### GDELT NLP Data Decision

GDELT Event CSV files are used as the stable data source for the NLP component. `SOURCEURL` slugs are extracted as lightweight textual representations of news articles. Event-level records are aggregated to article-level records by `media_date`, `SOURCEURL`, and `url_slug_text`.

### Weak Labeling Risk

A first weak-labeling attempt using both URL risk terms and GDELT event attributes produced a strict positive rate of about 52% and a broad positive rate of about 81%. Because URL terms are also used as NLP input, future weak labels should rely primarily on GDELT structured event attributes rather than URL keywords to avoid circular labeling.

### NLP Classifier Result

A weakly supervised TF-IDF Logistic Regression classifier was trained on January 2024 maritime URL slug text. It achieved ROC-AUC 0.915 and PR-AUC 0.864 against the weak target label. Positive terms included `houthi`, `red sea`, `attack`, `missile`, and `strike`, while generic maritime terms such as `port`, `logistics`, `cargo`, and `shipping` were associated with the negative class.

### 2024H1 Event Feature Expansion

The GDELT maritime event pipeline was expanded from January 2024 to 2024H1. It produced 44,005 article-level records and 26 weekly event-feature rows. The weak target positive rate was approximately 29.0%. Weekly event features include article counts, average NLP risk score, high-risk article share, total mentions, and tone indicators.

### 2024 Full-Year Event Signal Check

The event-feature coverage was expanded to the full year 2024. After merging weekly PortWatch operational data with weekly GDELT-derived event features, the 2024 dataset contained 53 weekly observations, including 8 abnormal next-week activity cases. Simple global event signals did not become stronger before abnormal weeks; average NLP risk score, high-risk article share, and total mentions were lower before abnormal weeks than before normal weeks.

### Multi-Scale Signal Check

Global, Europe-level, and local Netherlands/Rotterdam-oriented event signals were compared for 2024. The same-week signal comparison did not provide clear support for the simple hypothesis that higher maritime disruption news risk directly precedes abnormal Rotterdam port activity. Local average tone was more negative before abnormal weeks, but the local sample was small and noisy. This motivates expanding the temporal coverage and testing lagged or rolling event-signal structures before making any substantive research claim.

### 2023 GDELT Expansion

The GDELT event pipeline was expanded to 2023. One daily file, `20230323.export.CSV.zip`, returned 404 and was skipped. The resulting 2023 dataset contains 305,114 filtered event-level rows, 60,081 article-level rows, and 53 weekly event-feature rows. Event risk signals increased strongly in December 2023, which should be checked against PortWatch weekly abnormal activity before making any modeling claim.

### 2023 Event-Port Alignment Check

After merging 2023 weekly GDELT event features with Rotterdam PortWatch weekly activity, the dataset contained 52 matched weeks and 8 abnormal next-week activity cases. Compared with normal weeks, abnormal-next-week cases had higher average NLP risk score, higher high-risk article share, more total mentions, and more negative average tone. This provides preliminary directional support for the event-augmented logic, but the result may be partly driven by the December 2023 Red Sea disruption period and needs validation on additional years.

### 2025 Event-Port Alignment Check

The 2025 merged dataset contained 50 matched weeks and 10 abnormal next-week activity cases. Unlike 2023, article counts, average NLP risk score, high-risk article share, and total mentions were slightly lower before abnormal weeks. However, average tone and minimum tone were more negative before abnormal weeks. This suggests that simple event-volume or risk-probability features are not consistently aligned with Rotterdam weekly disruptions, while negative tone may still contain useful information. Multi-year and lagged-feature checks are required before modeling.

### 2023-2025 Multi-Year Signal Check

The 2023-2025 merged dataset contains 155 matched weekly observations and 26 abnormal next-week activity cases. Overall, abnormal-next-week cases show slightly higher average NLP risk score, high-risk article count, high-risk article share, and more negative tone, but total mentions are lower. Year-by-year results are inconsistent: 2023 strongly supports the event-signal logic, while 2024 and 2025 do not consistently support classifier-based event-volume signals. Negative tone is the most stable directionally useful signal, especially in 2023 and 2025. The next methodological step is to test lagged and rolling event-signal features.

### Lagged Event-Signal Check

Lagged and rolling event-signal features were tested on the 2023-2025 merged dataset. After correcting the missing-value handling, the lagged dataset contained 152 weekly observations and 26 abnormal next-week activity cases. The strongest directional differences were current-week average tone, current-week high-risk article share, two-week rolling average tone, and current-week minimum tone. Lag-1 and lag-2 versions were generally weaker. Since the target is next-week abnormal activity, current-week event signals remain temporally valid predictors.

### First Event-Augmented Modeling Result

A first temporal validation used 2023-2024 for training and 2025 for testing. The operational-only Logistic Regression model achieved ROC-AUC 0.813 and PR-AUC 0.421, while the operational-plus-event model achieved ROC-AUC 0.675 and PR-AUC 0.329. At the default 0.5 threshold, the event-augmented model improved F1 from 0.133 to 0.235, recall from 0.100 to 0.200, and precision from 0.200 to 0.286, but this improvement came with worse ranking metrics. This suggests that the current event signals affect the classification boundary but do not yet improve robust out-of-sample ranking performance.

### Threshold Sensitivity Result

Threshold sensitivity analysis confirmed that the operational-only model is stronger under the current setup. On the 2025 test year, the operational-only model achieved its best F1 of 0.563 at threshold 0.25, with recall 0.900 and precision 0.409. The operational-plus-event model achieved its best F1 of 0.429 at threshold 0.15, with recall 0.600 and precision 0.333. This indicates that the current NLP event features do not improve the best achievable classification trade-off, even though some event coefficients are directionally meaningful.

### Two-Stage Event Correction Result

A two-stage correction framework was tested in which the operational baseline first produced a baseline probability, and the second-stage model used this baseline probability together with NLP-derived event signals. Under temporal validation with 2023-2024 as training data and 2025 as test data, the two-stage event-corrected model improved default-threshold F1 from 0.133 to 0.421, recall from 0.100 to 0.400, precision from 0.200 to 0.444, and PR-AUC from 0.415 to 0.476. Its ROC-AUC was lower than the operational baseline, and its best threshold F1 remained slightly below the tuned operational baseline. This suggests that NLP event signals are most useful as a correction layer for positive-case detection rather than as a simple additive feature block.

### Engineered NLP Feature Result

NLP-derived event signals were transformed into relative event-pressure features, including rolling z-scores, spike flags, tone-drop indicators, and a composite event pressure index. Under the same 2023-2024 training and 2025 testing setup, the operational-plus-engineered-NLP model improved default-threshold F1 from 0.133 to 0.250 and recall from 0.100 to 0.200, while ROC-AUC and PR-AUC declined relative to the operational baseline. Positive coefficients for `risk_spike_flag`, `min_tone_drop_z4`, and `risk_score_z4` suggest that engineered NLP signals help identify some abnormal weeks, but they do not yet improve overall ranking performance.

### Two-Stage Error Analysis

Error analysis on the 2025 test year showed that the two-stage NLP correction changed 8 weekly predictions at the default threshold. It converted 3 operational-baseline false negatives into true positives, including 2025-06-02, 2025-09-01, and 2025-12-22, and converted 2 baseline false positives into true negatives. It also introduced 3 new false positives. This indicates that NLP event signals can meaningfully correct some baseline errors, especially around weeks with stronger negative tone or elevated high-risk article share, but the correction layer still requires calibration to avoid extra false alarms.

### 2022 GDELT Expansion

The GDELT event pipeline was expanded to 2022 as the first step toward a longer event-augmented dataset. One daily file, `20221110.export.CSV.zip`, returned 404 and was skipped. The resulting 2022 dataset contains 220,779 filtered event-level rows, 44,587 article-level rows, and 53 weekly event-feature rows. Early 2022 shows elevated NLP risk around January, which should be checked against PortWatch abnormal activity before integration into the multi-year model.

### 2022 Event-Port Alignment Check

After merging 2022 weekly GDELT event features with Rotterdam PortWatch weekly activity, the dataset contained 52 matched weeks and only 4 abnormal next-week activity cases. Compared with normal weeks, abnormal-next-week cases had lower NLP risk score, lower high-risk article share, fewer mentions, and less negative tone. The 2022 result does not support the event-signal hypothesis and suggests that some abnormal Rotterdam activity weeks may reflect local or operational fluctuations not captured by global maritime news signals.

### 2022-2025 Two-Stage Modeling Result

After adding 2022 to the training period, the combined dataset contained 207 weekly observations from 2022-2025. Training used 2022-2024 and testing used 2025. The two-stage event-corrected model improved PR-AUC from 0.469 to 0.506, but its default-threshold F1 declined from 0.600 to 0.500 relative to the operational baseline. The best-threshold F1 was also lower for the event-corrected model. This indicates that NLP event signals still add some ranking information for rare abnormal weeks, but their classification benefit is sensitive to the inclusion of years such as 2022 where port activity drops are not strongly aligned with global maritime news signals.

### 2021 GDELT Expansion

The GDELT event pipeline was expanded to 2021 with no skipped daily files. The resulting 2021 dataset contains 240,476 filtered event-level rows, 50,591 article-level rows, and 53 weekly event-feature rows. The 2021 weekly event features include several risk and tone spikes, and should be checked against Rotterdam abnormal activity before constructing the final 2021-2025 event-augmented dataset.

### 2021 Event-Port Alignment Check

After merging 2021 weekly GDELT event features with Rotterdam PortWatch weekly activity, the dataset contained 52 matched weeks and 7 abnormal next-week activity cases. Average NLP risk score was nearly identical between normal and abnormal-next-week cases, while high-risk article share and total mentions were slightly lower before abnormal weeks. This suggests that the current global maritime NLP signal was mostly neutral for 2021 Rotterdam abnormal activity.

### 2021-2025 Two-Stage Modeling Result

The final five-year event-augmented dataset contains 259 weekly observations from 2021-2025. Training used 2021-2024 and testing used 2025. At the default threshold, the two-stage event-corrected model improved F1 from 0.500 to 0.552, recall from 0.700 to 0.800, and precision from 0.389 to 0.421 relative to the operational baseline. However, ROC-AUC declined from 0.815 to 0.793, PR-AUC declined slightly from 0.471 to 0.460, and the operational baseline still achieved a higher best-threshold F1. This supports a nuanced conclusion: NLP-derived event signals can improve default-threshold abnormal-week detection in a correction-layer setup, but they do not yet provide a robust improvement in ranking performance.

### Multi-Scale Event Feature Construction

Article-level GDELT-NLP outputs from 2021-2025 were combined to construct global, Europe-level, and local Netherlands/Rotterdam-oriented weekly event features. The combined article-level dataset contains 287,504 rows, including 30,779 Europe-signal rows and 2,472 local-signal rows. The resulting multi-scale weekly event dataset contains 260 weekly observations and 28 columns. This enables testing whether spatially aligned event signals improve the two-stage correction framework compared with global signals alone.

### Multi-Scale Two-Stage Modeling Result

Multi-scale event correction was tested using 2021-2024 for training and 2025 for testing. Compared with the operational baseline, the multi-scale correction model improved default-threshold F1 from 0.500 to 0.593, recall from 0.700 to 0.800, precision from 0.389 to 0.471, ROC-AUC from 0.815 to 0.838, and PR-AUC from 0.471 to 0.505. Europe-only correction achieved the highest PR-AUC at 0.515 but lower F1 than the multi-scale model. This is the strongest evidence so far that NLP-derived event signals become more useful when spatially aligned across global, regional, and local layers rather than used as a single global signal.

### Multi-Scale Threshold Sensitivity Result

Threshold sensitivity analysis confirmed that the multi-scale correction result is not only a default-threshold artifact. The operational baseline achieved its best F1 of 0.581 at threshold 0.45, with recall 0.900 and precision 0.429. The multi-scale two-stage model achieved a higher best F1 of 0.600 at the same threshold, with recall 0.900 and precision 0.450. Its confusion matrix had the same number of false negatives as the operational baseline but one fewer false positive. This provides the strongest current evidence that spatially layered NLP event signals can improve the operational baseline.
